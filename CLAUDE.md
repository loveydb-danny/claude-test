# CLAUDE.md

이 파일은 Claude Code (claude.ai/code)가 이 저장소에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

썸원(SumOne) 커플 앱을 홍보하는 랜딩 페이지입니다. Apple 디자인 스타일을 적용한 세련된 단일 페이지 웹사이트로, HTML, CSS, JavaScript만으로 구성된 정적 페이지입니다.

### 썸원 앱이란?
- **타입**: 커플 다이어리 앱
- **핵심 기능**: 매일 하나의 질문을 커플에게 제공하고, 양쪽 모두 답변해야 다음 질문이 배송되는 구조
- **사용자 수**: 전 세계 1,000만+ 가입자, 100만+ 일일 활성 사용자
- **특징**: 반려몽 캐릭터 성장 게임 요소 포함

## 프로젝트 구조

- `index.html` - HTML 구조, CSS 스타일, JavaScript 로직이 모두 포함된 단일 파일
- `CLAUDE.md` - 프로젝트 가이드 (이 파일)

## 페이지 구조 및 섹션별 설명

### 1. Hero 섹션 (424-441줄)
**목적**: 방문자의 시선을 즉시 사로잡고 핵심 가치 전달

**구성 요소**:
- **메인 헤드라인**: "매일 더 가까워지는 우리의 이야기"
  - 감성적이고 직관적인 메시지로 관계 개선이라는 핵심 가치 전달
- **서브 헤드라인**: "바쁜 하루에도 단 한 번의 질문으로 더 깊어지는 관계를 경험하세요"
  - 낮은 진입장벽(단 한 번의 질문)과 명확한 혜택(더 깊어지는 관계) 강조
- **CTA 버튼**: Play Store 다운로드 링크
  - 눈에 띄는 흰색 버튼으로 즉각적인 행동 유도
- **통계 섹션**: 1,000만+ 사용자, 100만+ 일일 활성 사용자
  - 사회적 증거(Social Proof)를 통한 신뢰도 구축

**디자인 특징**:
- 그라데이션 배경 (#667eea → #764ba2)
- 움직이는 도트 패턴 애니메이션으로 생동감 부여
- 페이드인 업 애니메이션 (fadeInUp)

### 2. Problem 섹션 (443-470줄)
**목적**: 타겟 고객의 문제점 공감 및 심리적 자극

**4가지 핵심 고민**:
1. **대화 소재 부족** (💬)
   - 문제: 매일 만나도 대화가 단조롭고 반복적
   - 심리: 관계의 권태감, 소통 부족의 불안감

2. **바쁜 일상** (⏰)
   - 문제: 일과 공부로 상대방과의 시간 부족
   - 심리: 관계 우선순위 밀림에 대한 죄책감

3. **깊은 대화의 어려움** (📝)
   - 문제: 속마음 나누기가 어렵고 질문 방법을 모름
   - 심리: 감정 표현의 미숙함, 더 친밀해지고 싶은 욕구

4. **관계가 식어감** (💔)
   - 문제: 초반의 설렘 사라짐, 서로에 대한 무관심 증가
   - 심리: 관계 파탄에 대한 두려움, FOMO

**디자인 특징**:
- 4개의 카드 그리드 레이아웃
- 호버 시 살짝 올라가는 애니메이션 (translateY)
- 회색 배경 (#f5f5f7)으로 Hero 섹션과 대비

### 3. Features 섹션 (472-500줄)
**목적**: 앱의 핵심 기능을 차별화된 가치로 설명

**3가지 주요 기능**:

**a) 매일 하나의 질문** (477-483줄)
- **핵심 가치**: 낮은 진입장벽, 부담 없는 소통
- **심리적 효과**:
  - "한 줄이라도 괜찮아요" → 완벽주의 부담 제거
  - "작은 습관이 관계를 단단하게" → 꾸준함의 힘 강조
- **이모지**: 💌 (편지, 따뜻한 소통)

**b) 둘 다 답해야 다음 질문** (485-491줄)
- **핵심 가치**: 강제적 참여 구조, 상호작용 보장
- **심리적 효과**:
  - "혼자만 열심히 하는 다이어리는 금방 지쳐요" → 기존 솔루션의 한계 지적
  - "서로가 서로의 동기" → 상호 의존성, 함께하는 즐거움
- **이모지**: 🔒 (잠금, 양방향 참여 필수)

**c) 함께 키우는 반려몽** (493-499줄)
- **핵심 가치**: 게임 요소(Gamification), 지속 동기 부여
- **심리적 효과**:
  - "작은 성취감과 즐거움" → 보상 시스템
  - "게임처럼 재미있게, 하지만 더 의미 있게" → 재미와 의미의 균형
- **이모지**: 🐾 (캐릭터, 성장)

**디자인 특징**:
- 지그재그 레이아웃 (row-reverse)으로 시선 유도
- 큰 이모지 비주얼 (120px) + 그라데이션 배경
- 텍스트와 비주얼의 1:1 비율

### 4. How It Works 섹션 (502-524줄)
**목적**: 사용법을 간단명료하게 설명하여 진입장벽 낮추기

**3단계 프로세스**:
1. **앱을 열어요** - 질문 확인
2. **답변을 남겨요** - "한 줄이라도 괜찮아요" 재강조
3. **상대방을 기다려요** - 양방향 참여 구조 설명

**디자인 특징**:
- 숫자가 들어간 원형 배지 (step-number)
- 3단계만 보여줘서 복잡하지 않음을 강조
- 그리드 레이아웃으로 깔끔하게 정렬

### 5. Testimonials 섹션 (526-548줄)
**목적**: 사회적 증거(Social Proof)를 통한 신뢰 구축

**3가지 사용자 후기**:
1. **3년차 커플** - "하루가 더 행복해졌어요"
   - 장기 커플의 관계 유지 효과
2. **신혼부부** - "자연스럽게 깊은 대화를 나눌 수 있어요"
   - 대화 소재 제공 효과
3. **원거리 연애** - "서로를 더 생각하게 됐어요"
   - 거리를 극복하는 소통 도구

**디자인 특징**:
- 큰 따옴표 장식 (quote-mark)
- 회색 배경 카드로 내용 강조
- 3개 그리드 레이아웃

### 6. Final CTA 섹션 (550-555줄)
**목적**: 최종 전환(다운로드) 유도

**구성**:
- **헤드라인**: "오늘부터 시작하세요" - 즉각적인 행동 촉구
- **서브**: "1,000만 커플이 경험한 특별한 변화" - 다시 한 번 사회적 증거
- **버튼**: "무료로 시작하기" - 금전적 부담 제거

## 디자인 시스템 (Apple 스타일)

### 색상 팔레트
- **프라이머리 그라데이션**: `#667eea` → `#764ba2` (보라-퍼플 계열)
  - 로맨틱하고 감성적인 느낌
  - Hero, CTA, 버튼, 비주얼 요소에 일관되게 사용
- **텍스트 색상**:
  - 주요 텍스트: `#1d1d1f` (거의 검정, Apple 스타일)
  - 부연 텍스트: `#6e6e73` (회색)
- **배경 색상**:
  - 흰색: `#ffffff`
  - 라이트 그레이: `#f5f5f7` (섹션 구분용)

### 타이포그래피
- **폰트**: `-apple-system, BlinkMacSystemFont` (시스템 폰트)
  - 네이티브 느낌, 빠른 로딩
- **크기 체계**:
  - Hero 타이틀: 64px (모바일: 40px)
  - 섹션 타이틀: 48px (모바일: 36px)
  - Feature 타이틀: 40px (모바일: 32px)
  - 본문: 20px (모바일: 18px)
- **Letter spacing**: -0.02em (타이틀에 적용, 타이트한 느낌)
- **Line height**: 1.1~1.6 (가독성 고려)

### 간격(Spacing)
- **섹션 패딩**: 120px (모바일: 기본값)
- **요소 간 Gap**: 40px, 60px, 80px (일관된 리듬)
- **여백**: 넉넉한 공간 활용 (Apple의 미니멀 철학)

### Border Radius
- **버튼**: 980px (완전한 원형, Pill 스타일)
- **카드**: 20px (부드러운 모서리)

### 애니메이션
1. **fadeInUp** (59-68줄)
   - Hero 콘텐츠 등장 애니메이션
   - Y축 30px 이동 + 페이드인

2. **floatPattern** (47-50줄)
   - Hero 배경 도트 패턴 움직임
   - 20초 무한 반복, 미묘한 생동감

3. **Scroll Animation** (410-419줄, 557-576줄)
   - IntersectionObserver 사용
   - 뷰포트 진입 시 fade-in 클래스 추가
   - 스크롤에 따른 순차적 등장 효과

4. **Hover Effects**
   - 버튼: scale(1.05) + 그림자 확대
   - 카드: translateY(-5px)
   - 부드러운 0.3s transition

### 반응형 디자인 (361-407줄)
**브레이크포인트**: 768px

**모바일 최적화**:
- 폰트 크기 축소 (64px → 40px 등)
- Feature row를 세로 배치 (flex-direction: column)
- Gap 축소 (80px → 40px)
- 통계 섹션 간격 조정

## JavaScript 기능 (557-591줄)

### 1. Scroll Animation (558-576줄)
```javascript
const observerOptions = {
    threshold: 0.1,        // 10% 보이면 트리거
    rootMargin: '0px 0px -50px 0px'  // 하단 50px 여유
};
```
- **IntersectionObserver API** 사용
- `.fade-in` 클래스 요소 관찰
- 뷰포트 진입 시 `.visible` 클래스 추가 → CSS transition 발동

### 2. Smooth Scroll (578-590줄)
- 앵커 링크 클릭 시 부드러운 스크롤
- 현재 페이지에는 앵커 링크가 없지만 확장성 대비

## 개발 방법

### 로컬 개발
브라우저에서 `index.html`을 직접 열거나, 로컬 서버를 실행하세요:

```bash
# Python 3 내장 서버 사용
python3 -m http.server 8000

# 또는 Node.js http-server 사용
npx http-server -p 8000
```

그 다음 브라우저에서 `http://localhost:8000` 접속

#### 섹션 추가
새로운 섹션을 추가하려면:
1. `<section>` 태그로 감싸기
2. `section` 클래스의 패딩(120px) 자동 적용
3. 배경색 지정 (흰색 또는 `.problem` 클래스의 회색)
4. fade-in 애니메이션 원하면 `.fade-in` 클래스 추가

```html
<section class="new-section">
    <h2 class="section-title">새 섹션 제목</h2>
    <p class="section-subtitle">부제목</p>
    <!-- 내용 -->
</section>
```

## 심리학적 설득 전략

### 1. 문제-해결 프레임워크
- **문제 제기** (Problem 섹션) → 공감대 형성
- **해결책 제시** (Features 섹션) → 가치 전달

### 2. 사회적 증거 (Social Proof)
- 1,000만 사용자 통계 (Hero, Final CTA)
- 실제 사용자 후기 (Testimonials)

### 3. FOMO (Fear of Missing Out)
- "관계가 식어가는 것 같아요" (Problem 카드)
- "1,000만 커플이 경험한 변화" (Final CTA)

### 4. 손실 회피 (Loss Aversion)
- "처음의 설렘은 어디로 갔는지"
- "점점 서로에게 무관심해지는 것"

### 5. 낮은 진입장벽 강조
- "한 줄이라도 괜찮아요" (반복 강조)
- "무료로 시작하기"
- "3단계면 충분합니다"

### 6. 즉각적 행동 유도 (CTA)
- "지금 시작하기"
- "오늘부터 시작하세요"
- 버튼 2번 배치 (Hero, Final)
